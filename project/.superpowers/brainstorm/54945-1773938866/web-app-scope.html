What should the web app be?

The mobile app is a personal GTD client. The web app could serve a different role — or the same one on a bigger screen. Which direction fits your vision?

A

Full GTD Web App

Same features as mobile — Inbox, Next Actions, Projects, Waiting For, Weekly Review — but adapted for desktop with sidebar navigation. A complete standalone experience.

B

Admin / Dashboard Portal

Focus on overview, analytics, and management — not day-to-day task processing. Weekly review, project health, productivity trends. The "command center" while mobile handles capture and processing.

C

Hybrid — GTD + Admin

Full GTD functionality like the mobile app, plus admin features (user management, dashboard analytics) that only make sense on desktop. Best of both worlds, but bigger scope.

Context

Your current web project already has: sidebar layout, Dashboard view, Inbox view (stub), Users list/detail views, login/signup/OTP flows. The backend has full REST APIs for all GTD entities + dashboard + user management.
